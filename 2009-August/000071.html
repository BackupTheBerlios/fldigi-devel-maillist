Gentlemen,

Thank you all for the excellent suggestions. I think that DM780 is the place
to start, since they decoders are well packaged there.

We will do all the development necessary for this project.

We were hoping to be able to incorporate the code directly in our
application, since we already have a number of decoders already implemented
for other non-Ham signals. In fact, we have a DSP framework for decoders,
and we wanted to be able to take the decoders from the DM780 distribution
and place them directly in this framework. We realized that this would
actually require only modest modifications.

The decode server approach will take a little more work, and performance
will suffer some, but our customers can always buy a faster PC. On the other
hand, we are enthusiastic about contributing to the community.

Unfortunately, we license patented source code from several sources for some
special technology, so our commercial products will have to remain that way.

It may be a few weeks before we get started. When we get started, I will
send you our plans to see if there are ways we can make our efforts
applicable to a wider audience.

-Sam-

I should point out that some work has already been done on "headless"
operation.  Probaby not lean enough for general-purpose embedding but
maybe useful for testing.

If you build fldigi with --enable-benchmark, the binary will recognise a
few additional switches that can be used for timing the modems' decoding
performance non-interactively and, incidentally, for batch decoding of
audio files.  Fldigi will not display any windows but will still require
a windowing system.  This is unfortunately difficult to rectify but easy
to work around with X11.

There is also the possibility of controlling a "headless" fldigi via
xmlrpc.  The decoder output (text only) is obtained by polling the
xmlrpc server, or (IIRC) by connecting to the ARQ port.

There are some examplesin the manual page.


Update:  I see that batch decoding with output to a text file is broken
in 3.12; fixed in git.
