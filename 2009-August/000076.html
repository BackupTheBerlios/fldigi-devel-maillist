I have confirmation from Elecraft that no response to FAxxxxxx; is expected.
So the SETFREQ must not have an OK clause.  My attached K3.xml file is
correct in that respect.

Leigh/WA5ZNU
> I've found the reason for retry pauses with K3.xml on QSY.
>
> The SETFREQ command says it expects a FREQ response but no such response
> is forthcoming.
> I deleted the <OK>FREQ<OK> element from SETFREQ and the pauses have gone
> away.
> I have a confirmation message in to Elecraft to make sure this is the
> correct behavior and.
>
> The BW command response format is different in K2 and K3, mostly because
> the K3 offers finer control.
> I changed the BW command to enter K2 compat mode.  I'm not sure
> where/when fldigi is asking for the filter bandwidth, but I see failure
> logs in the event log without this change.
>
> I changed also to use K3 mode instead of K2 mode by default.
>
> My file is attached.
> I will continue to test these changes, but you may wish to pass this
> file on to other K3 testers.
