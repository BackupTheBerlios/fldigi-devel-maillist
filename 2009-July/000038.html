    MT63 transmit bug fix
    
        1. MT63 tx buffer was not correctly initialized at the start
           of each transmission.
        2. Added Rx Integration length selector to MT63 configuration.
        3. Added lowest / highest preamble tones as a signal acquisition
           aid.  User can enable lowest, both or none.

This bug has been with since the original conversion of gMFSK modem 
structures to fldigi modem classes.  The bug caused erroneous characters 
to be transmitted before the onset of the desired text.

This fix is important enough to be retrofit to the 3.12 release.
