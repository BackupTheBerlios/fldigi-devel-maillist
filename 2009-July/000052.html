The problem seems to be that I am using the Virtual Serial Port Emulator
software to allow multiple programs such as KTune and KComm to control the
K3 at the same time as Fldigi. Something has changed in the latest version
of Fldigi so that it will no longer work with VSPE. Not even Hamlib will
work with the virtual port, even if I close down all the other programs that
were using it. If I close VSPE and set up Fldigi to use the real serial port
then it works with either RigCAT or Hamlib.

This is probably of interest to anyone who uses KTune and/or KComm as I
describe using VSPE to allow two or more programs to control the radio on my
website. With Fldigi 3.12 you can't do that. I don't know if it is a matter
of configuring VSPE differently or whether there is any other workaround,
but that seems to be the situation at the moment. :(
