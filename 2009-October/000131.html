Thanks for your comments Stephane.  As it happens I've been using
valgrind --tool=callgrind and the very nice kcachegrind tool (which I
understand can also display oprofile's output), and fldigi was indeed
built with -O2 -g.  The waterfall fft processing is certainly a hotspot,
in fact it's hotter than some modems :-)

As for the SIMD stuff, I have only tested it in microbenchmarks which
are of course not the same as the real thing.  Once fftw3 is in place
(doing ffts out of place :) I will see if there are any easy to
vectorise hotspots.  And fldigi has a useful build switch
(--enable-benchmark) to test decoding speed with.
