Chris,  commit da2cc... might obviate the need to have an <RSID> macro 
tag.  Stelios is quite correct in his observation regarding such a tag.  
The macros are all evaluated and expanded before transmit.  The current 
macro parser would not allow both an <RSID-ON> and a <RSID-OFF> to occur 
in the same macro.  The last one parsed would determine the state.  That 
situation would not occur with the xmlrpc interface as each xmlrpc 
string is evaluated independently.  You could of course turn RSID on and 
then off before transmitting anything.
