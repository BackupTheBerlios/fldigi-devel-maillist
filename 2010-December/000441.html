I find that if I try to edit a macro in the upper macro bar then the name I
give it shows up in the lower macro bar (in the corresponding horizontal
position), and not in the correct position in the upper macro bar. If I
refresh the upper macro bar, by going through alt 2,3 etc it ends up showing
the correct name.

The name on the lower macro bar goes away after I open that lower macro for
edit and close it again. The actual contents of the macro on the lower bar
are correct, it is just that the screen seems not to be refreshing
correctly.

I do like the two macro bars, and agree that it would be useful if there was
a mouse-over hint to show which keys to use, but of course all of
these gimmicks do take up additional cpu cycles, so should for lower powered
machines the gimmicks should be able to be toggled on and off.

Also on a linux machine alt+1 etc may be taken up by the system for
switching workspaces ( as they are on my Ubuntu) so the keystrokes for
changing macro bars needs to be configurable.(Yes I know I could reconfigure
the system keystrokes, but I find I change workspaces more frequently than
macro bars).
