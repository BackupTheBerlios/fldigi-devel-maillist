for Linux, OS X, Win and Puppy

Known problem reported on 3.20 branch but applicable.  Do not exceed 100 
characters (including end-of-line chars) in the logbook Notes field or 
you will crash.  I am testing a fix on the 3.20 branch and will forward 
port when testing concluded.

3.21.0 development continues on Signal Browser.  Note that the 
configuration tab for the browser has been moved to the UI area.  Under 
UI/General there is a new check button that will allow you to restore 
the mouse wheel behavior to the macro buttons.  The default is OFF.

Ed, W3NR, has done some pre-testing on the RTTY signal browser.  It 
seems to work pretty well, but the upcoming RTTY contest should prove 
whether it has any value.

I am still chasing the PSK signal browser rainbow.  This version may (or 
may not) work better for you.  Be sure to try various settings of your 
audio input level and the browser squelch level and let me know if you 
see interaction between the two.

Changes from AR

PSK browser update
  * Changed linking of WF select to browser select
  * Added logic to allow squelch on/off button control of
    character updates to browser
  * Fixed --wfall-only seg fault

RTTY viewer
  Signal processing for multi-channel RTTY decoding

Macro Button Bars
  * Corrected button label updates after edit
  * Restored "macro set" buttons on button bar
  * Restored mouse-wheel action on button bars
    added configuration option to turn enable
    this feature

CONTESTIA
  Shortened ADIF reporting name to CONTESTI.
