 From Stelios:

I really think we should do a beta release this time, for several reasons:

1) The alpha team are good but we can never expect them to catch everything,
   e.g. see the 3.12 exceptions on win32.

2) There is so much more new stuff in 3.13 that I would expect equally more
   uncaught release bugs.

3) All our new (non-maintenance) releases have actually been betas, just not
   labelled as such.  This raises expectations and results in disappointment
   when serious issues are found, and it forces us to make more releases.

I agree with Stelios, but would like to hear from the other developers.

A.  Agree / disagree with a beta release
B.  Does the degree of change from the 3.12.5 release justify this as a 3.2 vice 3.13?
C.  Should OSS be removed from this release?
D.  Should Pulseaudio be the preferred audio i/o when the distro is set up that way?
