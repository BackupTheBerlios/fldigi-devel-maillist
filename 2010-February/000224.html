Comment (by sb42):

 The warning is emitted by the dynamic linker/loader, not fldigi. It is not
 a
 bogus warning since the ABI has actually changed.

 We don't have to use hamlib_version in fldigi but it's useful to have that
 information (compile- vs. run-time version) in the --build-info output.

 The best solution would be for hamlib to export the runtime version string
 via
 a symbol whose size will not change across versions (e.g. a function).
