Many of you have asked for advance notice on changes to the website
so I thought that I would take this opportunity.

There are two developments that I'd like to pre-announce to this group.

First, starting in May, the 'Detail' part of QRZ callsign listings will 
be available
only to those users who login and for whose browsers support both Javascript
and Cookies.   Users who do not login, or who have Javascript or Cookies
disabled will get only the first page, basic callsign data, i.e. name 
and address.

Secondly, we're going to expand our data interface to provide JSON as an
alternative to XML.   We're already using JSON in several places on the site
with great results.  The interface will be very similar to the XML with 
the same
types of session key exchanges, etc., but simply encapsulated with JSON
instead of XML.  Our XML will not be affected by this new feature.

As a quick example, using XML, we list our callsigns like this:

<Callsign>
<call>AA7BQ</call>
<dxcc>291</dxcc>
<fname>Fred L</fname>
<name>Lloyd</name>
....
</Callsign>

In JSON, this becomes:

{callsign: {
call: 'AA7BQ',
dxcc: '291',
fname: 'Fred L',
name: 'Lloyd',
....
}}

As you can seem there are some advantages to JSON however it really makes no
difference whether you use it or not, unless you're coding in Javascript 
where it's a whole
lot simpler to decode.
