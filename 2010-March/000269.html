Hi,

Many bug fixes pushed in the branch pu/rc :

* Less calls to REQ(update_rx_pix) to reduce lost pixels.
* Received image now starts at the good column when phasing detected.
* Fixed proportion of transmit image.
* Uses proper C_FIR_filter class.
* Corrected bug when slanting zoomed image.
* Zoomed-in images correctly displayed.
* No left shift of images when increasing height.
* Reception image can be zoomed in/out. Usual width is now 1809 pixels.
* Fix error on default window width.
* Slant adjustment is split in integer lpm value and slant percentage.
* Manual mode can continuously split into successive max size images.
* Clean function for converting ioc into width.
* Added lpm=240 value.
* Adjust reception window width.
* Maximum number of lines is now a Fl_Int_Input, easier to change.
* Code cleanup.
* Implemented a fully manual mode.
* Maximum number of lines is now editable.
* Fixed scroll error of reception image.
* Avoids crashes when choosing another wefax modem, while receiving.
* Now it compiles without Hamlib.
* std::cerr replaced by LOG_ERROR.

I tried to take into account all bugs and remarks.

Cheers

RC
