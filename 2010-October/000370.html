 Over several fldigi and MixW program changes, fldigi cannot reliably
 decode MixW olivia, although the reverse is no problem.  I presume (I am
 not a programmer) that there was a standard olivia description from which
 fldigi's decoder was developed - but it does not seem to be a standard.
 MixW to MixW decoding works as intended; MultiPSK can reliably decode
 MixW, and DM780 is usually successful, which is odd since they are said to
 have the same decoder as fldigi.
 It seems that some conferencing between the developers is in order, which
 is the only way this will get resolved.
