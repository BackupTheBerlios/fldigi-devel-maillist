Hi,

I am working on a very small project and need the s/n ratio for the 
current modem.

Just saw this method:

void modem::s2nreport(void)
{
	double s2n_avg = s2n_sum / s2n_ncount;
	double s2n_stddev = sqrt((s2n_sum2 / s2n_ncount) - (s2n_avg * s2n_avg));

	REQ(pskmail_notify_s2n, s2n_ncount, s2n_avg, s2n_stddev);
}

Apparently, a couple of modems update the members modem::s2n_sum, 
s2n_valid and s2n_ncount.

At the same time, many modems have this kind of code sequence:

double tmp_snr = the_signal.image_noise_ratio();
char snr_buffer[128];snprintf(snr_buffer, sizeof(snr_buffer), "s/n %3.0f 
dB",  tmp_snr );
put_Status1(snr_buffer);

Therefore I have these questions:
* What is the exact role of s2n_sum, s2n_valid and s2n_ncount.
* Should wefax update these variables too ?
* Why not putting the display of the snr in a modem member (Calling 
put_Status1). This would simplify the code.
* Based on the existing members, how can I get the current s2n ratio ?

Thanks in advance.
