Dave,

I have heard from Jozef regarding documentation for you and he
does not require an NDA.  However, he has one condition before
I forward you the internal protocol document.

> I have no problem if final source code is public, from source code is
> very hard if not impossible to make different, software compatible
> product. Of course if protocol document isn't 1:1 copied in the
> source files :-)

In other words, you are free to make the source public under the terms
of GPL but the protocol document (or even substantial parts of it) may
not be included in the source or released to others beyond the limited
number of persons with a direct need to know.

> All of the microHAM products are first class. I think that Josef is
> ignoring a large (and growing) part of the ham community by not
> providing Linux / OS X support.

OS-X seems to be covered adequately by Chen, Don Agro (MacLogger DX),
and Tom Lindner (DL2RUM).  Jozef has always believed that LINUX support
would be nice but he has been burned by the LINUX developer community on 
several previous occasions when promises of support have turned into
dead silence.

microHAM's programming resources are already stretched to the limit
in supporting the existing/planned hardware development and Windows user 
base.  From Jozef's perspective, LINUX is a very small minority market 
that does not justify the expense of adding another full-time
programmer to the development staff for what most estimates consider
to be at best 15% of the amateur market.

PS. USB Interface III should work in the LINUX environment without
     any special support software.  Both the sound card and USB to
     serial converter are standard hardware (no embedded processor
     to support) with manufacturer supplied drivers.  I have used
     it in both OS-X and Windows without Chen's uH Router or the
     microHAM Router package.
