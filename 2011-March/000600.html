Wrap File reception
      * Open NBEMS message folder if required by configuration
      * Start flmsg if required AND string "<flmsg>" is in the received 
text.

Configuration items have changed:



If you check "Open message folder" this event will be triggered by any 
"wrap" reception.

If you check "Open with flmsg" this event will be triggered only if a 
wrapped-flmsg file has been received.

If selected both triggers can occur on a received file.

The flmsg: entry must contain the absolute path-name to the flmsg 
executable.  Shown above for Linux.  The "Locate flmsg" button behaves 
differently on Linux, Windows, and OS X.  On Linux and Windows it opens 
a file finder dialog which allows you to identify and select the correct 
file.  On Windows this might something like "C:\Program 
Files\flmsg-1.1.1\flmsg.exe". 

On OS X it opens a "Finder" window preset to the Applications folder.  
You want to identify the target executable and NOT the app image.  To do 
this right click on the flmsg icon in the Applications folder and select 
"Show Package Contents".  Double click on Contents, and then on MacOS.  
The "Finder" should now be showing a simple executable icon named 
"flmsg".  Grab that icon and drag and drop it onto the "flmsg:" control.



All this work will enable fldigi to open a new instance of flmsg for 
every reception of a wrapped-flmsg data file.  This should work the same 
on all OS.  You should also be able to close the flmsg instances and/or 
fldigi independently.
