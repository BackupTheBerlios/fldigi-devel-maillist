    I see that auto opening in Flmsg and the browser is an either/or thing, 
that you can't have the check boxes to open in Flmsg and the browser both 
checked at the same time.  In other words, the controls act like HTML 
radio buttons rather than true checkboxes.

    I'm just thinking out loud here...I wonder if there's a case when we'd 
want to be able to auto open with both?  I'm thinking if I'm say a NCS, 
I'd want the message open in Flmsg for relay or replying to an ICS-213, 
and it would be great to have the message open in the browser at the same 
time.

    So, maybe we would want to be able to auto open in Flmsg and a browser 
at the same time.  What would be the drawback to doing this?
