Hi,

here is a first draft of new XML-RPC functions for sending and receiving 
faxes. (Extracted from the new html page).

wefax.state_string	s:n	Returns Wefax engine state (tx and rx) for 
information.	

wefax.skip_apt	s:n	Skip APT during Wefax reception. Returns an empty 
string if OK, or an error message.	

wefax.skip_phasing	s:n	Skip phasing during Wefax reception. Returns an 
empty string if OK, or an error message.	

wefax.set_tx_abort_flag	s:n	Cancels Wefax image transmission. Returns an 
empty string if OK, or an error message.	

wefax.end_reception	s:n	End Wefax image reception. Returns an empty 
string if OK, or an error message.	

wefax.start_manual_reception	s:n	Starts fax image reception in manual 
mode, so images will always have the maximum number of lines. Returns an 
empty string if OK, or an error message.	

wefax.set_adif_log	s:b	Set/reset logging to received/transmit images to 
ADIF log file. The boolean parameter is a flag which is false by 
default. Returns an empty string if OK, or an error message.	

wefax.set_max_lines	s:i	Set maximum lines for fax image reception. No 
files will have more lines, but may be shorter if the fax end is 
automatically detected. Returns an empty string if OK, or an error message.	

wefax.get_received_file	s:i	Waits for next received fax file, returns 
its name with a delay in seconds. Empty string if timeout. Returns a 
file name, or an empty string if all received files have been queried. 
They are returned in chronological order.	

wefax.send_file	s:si	Send file with a maximum delay in seconds. Returns 
an empty string if OK otherwise an error message. Note that sending a 
file take typically ten ,inutes. Progression can be followed on the GUI.

Any remark welcome.
