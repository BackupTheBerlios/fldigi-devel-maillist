Hi,

At the moment, still working on a Synop decoder for the Rtty modem, most 
of known token are decoded. It would be convenient to add a separate 
test program, working as a command-line tool:

* It allows to focus on the decoding itself (The syntax is sometimes 
complicated and it is crucial to reproduce similar input patterns).
* Apparently, having a separate program is an interesting feature.
* It could use exactly the same code implemented into fldig.

It would work like that:

flsynop inputfile1.txt inputfile2.txt ...

and would generate:

inputfile1.out : The input text with only the Synop groups decoded (The 
rest being strictly identical). This would be similar to the information 
displayed to the output panel of the GUI.

inputfile1.msg : Synop messages (And only them) extracted and properly 
formatted, reading for publishing.

What do you think ?
