Doxygen assumes all comments are one-line-above a variable/function/macro

To force inclusion of a comment at the end of a line one uses " ///< " instead
of  " // "
Comments at the end of line are ignored unless this source-code
modification is done.
   Source :  http://www.stack.nl/~dimitri/doxygen/docblocks.html

Over time these small changes can be done as they are only comments (and
therefore safe)

Am working on a script here that others can use to easily generate these
documents.
Note though...
   these docs are about 700MB in size (and that is per git-branch)
   each branch takes quite a while to generate, longer than compiling fldigi
   will keep the website as up-to-date as possible so everyone does not
have to generate the docs.

http://www.bitsandbytes.info/doxygen/3.22.0CH/<http://www.bitsandbytes.info/doxygen/3.22.0CH/classfax__implementation.html>
