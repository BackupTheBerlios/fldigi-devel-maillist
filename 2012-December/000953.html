  [patches that avoid eating the space following -lpthread]

I have two further questions.

1) Why is it ok/desirable to remove -lpthread?  Is it a bug in
xmlrpc-c-config to have placed it there?  It would be nice to have a
comment in xmlrpc.m4 to explain.

2) The sed pattern seems like it will not match "-lpthread$", and it
will match "-lpthreadfoo".

I would be tempted to suggest the following
  sed -E 's/(^| )-lpthread($| )/\1\2/
but unfortunately POSIX does not require that sed handle extended
regular expressions:
  http://pubs.opengroup.org/onlinepubs/9699919799/utilities/sed.html
and it seems | is missing in BREs, so the best I can come up with is

sed -e 's/^-lpthread$//' \
    -e 's/^-lpthread / /' \
    -e 's/ -lpthread$//' \
    -e 's/ -lpthread / /'
