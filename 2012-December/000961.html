For the archives, this wasn't a "NetBSD compile error".  It was a bug
in the sed script.  On NetBSD xmlrpc-c-config did not emit -lpthread, so
the bug was not triggered.   On Dragonfly, xmlrpc-c-config did have
-lpthread, and led to the mangled double-option.  gcc 4.7 correctly
fails with such options, whereas earlier gcc versions apparently
silently drop it.
