Hello Dave,

All the best for the New Year.

I tested version 3.22.0BM under Windows XP SP3.

I checked the following:

1. RX and TX of various PSK, THOR and MFSK modes with RxID and TxID: all
ok. Mode browser worked fine under PSK63.

2. I checked the config section. All looks fine and I made a few changes.

All working with one really minor issue: if Fldigi is in TUNE mode and a
change is made in the  audio config (I changed to MONO output in my case),
the TUNE is interrupted but the TUNE button remains ON. It works fine
otherwise.

3. Made one log entry. Saved and exported OK to text file.

I have not tested the rig control side.

So all in all, looking like the great Fldigi that I know.

> **
> I have selected you to participate in a special test of all of the flxxx
> applications, either because you are on the development team, a very active
> tester, a Linux distribution maintainer, or a user on OS-X or Puppy Linux.
>
> The maintainers are aware, and perhaps others as well, that the underlying
> user interface for the applications, FLTK (Fast Light Tool Kit) has dropped
> support for the FLTK-1.1 branch.  Future support will only be for the 1.3.x
> branch.  Future Linux distributions will not provide repository support for
> FLTK-1.1.  It was therefore necessary to port all of the applications to
> the FLTK-1.3.0 library.  A simple upgrade right?  Wrong!!  The 1.3.0
> version of FLTK now supports UTF-8 character encoding.  That alone touches
> nearly every widget in the UI set.  Worse, the changes made in the library
> code is different for Linux, Windows and OS X.  I think that the flxxx
> application base is now good to go.
>
> I've posted source code and installation files that have been built to the
> FLTK-1.3.0 library.
>
> http://www.w1hkj.com/trials-1.3.0/src/
>
> Windows installation files at
>
> http://www.w1hkj.com/trials-1.3.0/win/
>
> Puppy pet install files at
>
> http://www.w1hkj.com/trials-1.3.0/pet/
>
> and OS X universal disk image files at
>
> http://www.w1hkj.com/trials-1.3.0/osx/
>
> I've tested the source code for proper build against both the 1.1.10 and
> the 1.3.0 FLTK distributions.  The build script tests for the version of
> FLTK and then compilation is adjusted accordingly.
>
> I've tested the 1.1.10 build binaries on Ubuntu 10.04.
> I've tested the 1.3.0 build binaries on Debian squeeze.
>
> I've tested the OS X dmgs on both a G4 mini-mac (Motorola chip) and a
> G4-Intel machine.  Both load very fast, so the dmg is providing native
> code.  Let me know if you see otherwise.
>
> I've tested the Windows installation on Vista, but not XP or Win-7.
