> I downloaded all of the code and built all with fltk 1.1 on Ubuntu 11.04

All compiled fine and ran OK.

I then downloaded fltk 1.3 built it from source  and then rebuilt all seven
programmes in a different directory.

Again all appeared to compile OK, and run correctly.

The configurations screens appear to look correct in all programmes and
versions.

Must admit, I confused myself for a moment when I started a second fldigi
(not realizing I already had one running) and thought things were broken
and flrig was not talking to the second instance, (but was of course to the
hidden first one). Happily soon realised my error - before sending in an
erroneous  bug report.

I just hope that installing fltk 1.3 has not broken any other programme.

Will continue testing, including on Windows 7 64 bit as well, but these are
my early results.

So, all is good, I wish you Dave and the other developers an excellent New
Year ( and all lurkers).

> I have selected you to participate in a special test of all of the flxxx
> applications, either because you are on the development team, a very
> active tester, a Linux distribution maintainer, or a user on OS-X or
> Puppy Linux.
>
> The maintainers are aware, and perhaps others as well, that the
> underlying user interface for the applications, FLTK (Fast Light Tool
> Kit) has dropped support for the FLTK-1.1 branch.  Future support will
> only be for the 1.3.x branch.  Future Linux distributions will not
> provide repository support for FLTK-1.1.  It was therefore necessary to
> port all of the applications to the FLTK-1.3.0 library.  A simple
> upgrade right?  Wrong!!  The 1.3.0 version of FLTK now supports UTF-8
> character encoding.  That alone touches nearly every widget in the UI
> set.  Worse, the changes made in the library code is different for
> Linux, Windows and OS X.  I think that the flxxx application base is now
> good to go.
>
> I've posted source code and installation files that have been built to
> the FLTK-1.3.0 library.
>
> http://www.w1hkj.com/trials-1.3.0/src/
>
> Windows installation files at
>
> http://www.w1hkj.com/trials-1.3.0/win/
>
> Puppy pet install files at
>
> http://www.w1hkj.com/trials-1.3.0/pet/
>
> and OS X universal disk image files at
>
> http://www.w1hkj.com/trials-1.3.0/osx/
>
> I've tested the source code for proper build against both the 1.1.10 and
> the 1.3.0 FLTK distributions.  The build script tests for the version of
> FLTK and then compilation is adjusted accordingly.
>
> I've tested the 1.1.10 build binaries on Ubuntu 10.04.
> I've tested the 1.3.0 build binaries on Debian squeeze.
>
> I've tested the OS X dmgs on both a G4 mini-mac (Motorola chip) and a
> G4-Intel machine.  Both load very fast, so the dmg is providing native
> code.  Let me know if you see otherwise.
>
> I've tested the Windows installation on Vista, but not XP or Win-7.
>
> It's time for a beer :-)
>
> If you report back you might want to reply to all so that the other
> testers know your findings.
>
> 73, Dave, W1HKJ
