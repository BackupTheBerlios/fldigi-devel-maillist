OK, after much trial and error I have tracked down the config problem
causing X11 crashes on my Debian Squeeze AMD64 system. It was nothing to do
with the new features, but font configuration.

- I have used for a long time Tx and Rx pane fonts of Fixed Bold, size 15.

- This causes the crash in the 3.22.0CB and 3.21.43. There is no problem up
to and including 3.21.41

- Changing the font size in the config file to 16 prevents the crash

- Size 15 is available in the font configuration GUI. If this is selected
the font changes as expected for that session, but if the config is saved
and fldigi is restarted X11 crashes again.

- I tried size 14 (16 looks horrible on my system) and that also causes the
crash.
