Dave;
	Found a conflict with AH version of fldigi and CQRLog.  When operating
with CQRLog in remote mode (ctl-M) I tried to save a QSO. The time
fields were written with hours, minutes and seconds. Fldigi displayed
1922, saved 19:22:32 and CQRLog receives 192232!! Current version
(1.4.1) of CQRLog looks for only hours and minutes.

Seems we need some coordination with these 2 excellent programs hi hi.
