I used the K3 rigcat on 20M and both reading the VFO and using QSY 
worked on PSK31 on 20 meters. Setting the VFO via the green-led rig 
control in the main window worked.  I used 38400 baud, though default 
for KX3 is 4800.  3 retries / 250ms retry interval / 100ms write delay

I've noticed the K3 loses sync overnight, and refuses to respond to any 
RS232 commands, not just from fldigi, under reset.
I've experimented with various ideas, and it's vaguely possible it's my 
RS232 port, but it's a 4-port PCI card and the other ports work fine 
when this wedge happens.  I'll look for the same problem with the KX3.
