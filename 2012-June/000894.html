I use the virtual KISS interface in MixW ( MultiPSK also does this ) so that

Dave Kleber:

Here is the AX25 specification:   http://www.tapr.org/pub_ax25.html

It's just a protocol, and so requires no special hardware. AX25 is a special 
communications protocol developed for amateurs way back when, as Packet 
Radio was developed.

This intro to Packet Radio by Larry Kenny, WB9LOZ is much better than any 
explanation that I could offer here, or anywhere else. It explains about 
basic Packet. http://www.choisser.com/packet/

Advanced Packet intro: 
http://uspacket.org/network/index.php/topic,15.0.html

In general: cruise around the LIBRARY section at USPacket.Org for more info: 
http://uspacket.org/network/index.php?action=forum#2

Many learned and learned-sounding gentlemen ( usually amateur tcpip fans ) 
will go on at great length about how awful ax25 is, how it couldn't possibly 
work on HF, and how easy it would be to better it - but I have heard this 
little story hundreds of times over dozens of years - and guess what? - Not 
a single one of these learned-sounding gentlemens have managed to cough up 
anything that does the same thing at all, much less do it any better... - 
All hat and no cattle, as we say here in Texas.

Meanwhile, the "impossible" 300 baud HF Packet network that started a 
quarter of a century ago is still a going concern today.

Amazing, isn't it?

I would like to see more AX25'ed digital modes besides 300 baud Packet, some 
slower and some faster, and some better at cutting through the crud. - I'd 
like to have a virtual KISS interface ( software-only KISS, no hardware TNC 
required ) so that the newly available AX25 digital modes can be used by 
amateurs with a wide variety of popular Packet Radio software, already 
well-developed and ready to go.
