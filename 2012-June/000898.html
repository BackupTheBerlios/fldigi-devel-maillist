All,

Summary:  fldigi compiles and runs fine on FC15 x86-64
                 fldigi has tons of compile errors on FC17 x86-64

FC15 uses fltk 1.1 and gcc 4.6.3;
FC17 uses fltk 1.3 and gcc 4.7.0

Details of the FC17 environment:

The RPMs for fltk, fltk-devel, and fltk-fluid are installed:

   rpm -q fltk fltk-devel fltk-fluid
      fltk-1.3.0-4.fc17.x86_64
      fltk-devel-1.3.0-4.fc17.x86_64
      fltk-fluid-1.3.0-4.fc17.x86_64

The fltk-config utility runs, and options return reasonable values...

But the configure.log indicates that FLTK was not considered.....
The environment, set prior to running "./configure
--enable-optimizations=sse2 --with-xmlrpc --enable-debug":

FLTK_CONFIG=/usr/bin/fltk-config
FLTK_LIBS=/usr/lib64 -lfltk-images -lpng -lz ljpeg -lfltk -lXext -lxft
-lfontconfig -lXinerama -lpthread -ldl -lm -lX11
FLTK_CFLAGS=-I/usr/include -I/usr/include/freetype2 -D_LARGEFILE_SOURCE
-D_LARGEFILE64_SOURCE
    -D_THREADSAFE -D_REENTRANT

It matches the env for FC15, which gets set properly from ./configure

But ./configure issues a "not checking for FLTK"
at line 9801 of ./configure.

Has anyone else successfully compiled fldigi on FC17 x86-64?
