Hi,

New version and many fixes to the Synop decoder and KML display for 
Synop and Navtex.

On branch pu/rc

http://git.berlios.de/cgi-bin/gitweb.cgi?p=fldigi;a=commit;h=a433fcf26f0e8c9d8c7c341e8c9c2355aa0a13d8

Synop+Navtex+KML

* Removed names and email addresses from WMO stations file.
* Fixed Italian stations in Navtex stations file (Thanks Franco !).
* Speedup of KML files IO.
* Fixed WMO station names ambiguity (Conception in Latin America)
* KML placemarks names are shorter if possible.
* East longitude positive, West negative. Fixes bad Navtex detection.
* Better ship name detection vs WMO identifier.
* Better Synop decoding: 4Esss, 55jjj, 9SSss,
* Char '=' used as a Synop end-of-section marker.
* Added option command in KML tab executed on KML file save.
* Comments and code cleanup.
* Better filtering of XML forbidden chars in KML files.

Among other things, it is possible to start a shell command when KML 
files are saved. This allows for example to start automatically Google 
Earth, or call GPSBabel to convert KML files to GPS format GPX. Commands 
are not restarted as long as their process id is still there, which 
makes that GoogleEarth is restartd only if it crashes, but a GPSBabel 
conversion will be called each time.

Any idea/remark/critics warmly welcome. When things will be approved I 
will (try to) squash all related commits into a single one, ideally 
applicable to version 3.21.

Many thanks for those who have helped a lot with their tests and comments.
