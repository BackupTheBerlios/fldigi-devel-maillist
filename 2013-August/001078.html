Hi Dave,

For testing the MFSK and THOR soft-decoder improvements on my origin/pu/jp
branch I placed a few Test WAV files at http://www.bitsandbytes.info/audio/

The test files used in testing were generated using Linsim (snr) or
Audacity (fades/CWI).

The MFSK summary:
* MFSK decoder sensitivity improved noticeably with these code-changes,
especially when using acoustic-coupling.
* MFSK16   -- decode at -14db snr, or with one CWI
* MFSK128L -- decode at -6db snr, or with 1 CWI, or with multiple 3-second
complete-fades
* MFSK64L -- decode at -8db snr, or with 1 CWI, or with multiple 3-second
complete-fades

The THOR summary:
* The THOR 25,50,100 FEC generator change noticeably improved
error-correction capabilities.
* THOR100 with New generators can correct 2 CWI carriers of interference
* THOR100 with Old generators and 2 CWI carriers is no-decode
* The THOR soft-doppler change provided only slight soft-decoding gains.

Other Files:
randomtext.txt -- the ASCII input to generate the WAV files
generate-* .wav     -- the raw/perfect WAV files from fldigi before running
through linsim/audacity

Feedback and additional testing is appreciated on these changes. Hope to
make into the next release if possible.

yesterday evening.
