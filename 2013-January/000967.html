Hi Rainer-

I see that you actually (first) tried to add the "gwibber-daily" PPA to
your system, not the fldigi PPA.  I'm sure that its because the
Launchpad PPA system confusingly shows the gwibber-daily screenshot as
an example for *all* PPA's...  You're supposed to have replaced
gwibber-daily with the name of the actual PPA that you're trying to
install (gwibber has nothing to do with fldigi at all).

Specifically, this is the correct command sequence to install my fldigi
PPA:

$ sudo apt-add-repository ppa:kamalmostafa/fldigi
$ sudo apt-get install fldigi

All that said, it looks from the crash message like you did get 3.21.65
installed in the end, so the crash bears further investigation.

I recommend first that you try pushing your .fldigi directory aside to
test whether the crash is occurring due to some inconsistency in the
fldigi config files, as explained here:
https://lists.berlios.de/pipermail/fldigi-devel/2012-November/000937.html

Does starting with no $HOME/.fldigi/ directory help any?

73 de KA6MAL

 -Kamal
