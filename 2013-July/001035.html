This version fixes the Thor out-of-range, and uninitialized variables 
bug.  It also makes the event logging thread safe.  The thread safe 
logging should have been fixed many versions ago.  I've tested BK on 
Linux, OS X, XP and Win8.  I do not have a current version of Win7 or 
Vista which I can use for testing.

Added/enhanced command line debugging switches:

--debug-level <0..5>, 0 - QUIET, 1 - ERROR, 2 - WARN, 3 - INFO, 4 - 
VERBOSE, 5 - DEBUG
--debug-pskmail [no flag, if declared this enables additional event 
logging for the ARQ server]
--debug-audio [no flag, if declared this enables additional even logging 
for the sound interface to OSS, PortAudio, PulseAudio]

Windows users: you can set the command line switches by modifying the 
properties of the desktop launch icon for fldigi.  Right click on the 
icon, open the properties dialog, and then add the switch statements to 
the end of the contents of the "Target:" control.

I suggest testing with flarq / flamp using

--debug-level 5 --debug-pskmail

If no failures after 48 hours then you can test without those switch 
settings.

Report any failures direct to the return address.  Please include as 
much detail as possible, including the status_log.txt that was written 
to the fldigi files folder at the time of the failure.

The latest test version of flamp can be downloaded here: 
http://www.w1hkj.com/alpha/flamp/
