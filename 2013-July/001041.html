The gradual modification of existing Fldigi comments into Doxygen format
would be very helpful, and a low-risk set of changes (being comments only).
Am working on a "quick Doxygen formatting help-page" that will live in the
Doxygen HTML docs and explain, with examples, how each comment-type acts.

In order to make the modem classes appear higher up in the Doxygen docs,
each header file needs a Doxygen comment with the keyword "@class" in it.
Then Doxygen will generate a nice list of these now "documented classes".
As the code becomes more Doxygen-documented, the generated documentation
will continue to improve. What exists currently is Doxygen's
auto-generating from 99% undocumented code. (undocumented from Doxygen's
perspective: no tags, etc...)

Both the scripts really should have a "clean" action to safely cleanup
after themselves. Will definitely add that feature to the scripts.

Doxygen can do just about everything I am discovering, and the thickness of
the manual reflects this. hi hi
Will continue to work on implementing more comprehensive Doxygen into in
the header files.

Glad to see this is proving to be useful,
