On 07/23/2013 11:04 PM, John Phelps wrote:
> Hi all,

> The Fldigi Doxygen commit is completed / finalized / tested and has 
> been slightly modified.
> It can be pulled from my origin/pu/jp branch for those interested. 
> Feedback welcomed, especially if something is found missing / broken / 
> lacking.
>
> Quick summary of usage:
> * Requires an OSX or Linux / Unix system to execute.
> * To both generate Doxygen documentation and execute cppcheck 
> sourcecode analysis execute:
> scripts/doxygen/gen_doxygen_docs.sh run
> * To run only the cppcheck analysis tests execute:
> scripts/tests/cppcheck/gen_cppcheck_results.sh run
>
> Both of the scripts will show help/usage if no "action" parameter is 
> given.
> Available actions are:
> * run - Generate the docs / tests
> * clean - Cleanup all files generated by this script
> * help - Print explanation and usage then exit
>
> Just executing the scripts without any parameters will show a help and 
> usage summary.
>
> Am also in the process of updating the Fldigi Wiki with information on 
> using these new tools. To be finished soon.
> https://fedorahosted.org/fldigi/wiki/Documentation/HOWTO#Developmentsupporttoolsandscripts
>
> As errors cppcheck find are found to be erroneous, they can be 
> suppressed by putting the following comment on the line just-before 
> the erroring line:
> //cppcheck-suppress <type>
>
> Where <type> is the type of error to suppress (error, warning, 
> performance, etc...)
>
> I will work on getting a copy of the Doxygen docs up on my webserver, 
> then post a link here once finished uploading (may take overnight)
>
