Hi to all Fldigi developers,

I have created a new branch in which to develop the NBEMS-FM mode:
tmp/nbems-fm
Any and all and help with the project is appreciated.

My first push to the branch is code to extend the PSK modulator and
demodulator both from 4PSK (QPSK) to 16PSK.
Also there is test-code to re-allocate (temporarily) the existing PSK menu
items for easier testing of these modes.
* BPSK modes become 8PSK
* QPSK modes become XPSK (4psk)
* PSKR modes become 8PSK

Everything compiles and runs encodes/decodes properly. However 16PSK does
not decode well... yet hihi.

Here are some speed estimates for NBEMS-FM at different modulations:
1) 500baud 8PSK = 1500 bps
2) 500baud 8PSK * 2 carriers = 3000 bps
3) 500baud 8PSK * 4 carriers = 6000 bps
4) 125baud 8PSK * 16 carriers = 6000 bps

And... notes on the previous modes:
1) Tested on-air working. Decode good.
2) Tested on-air, but decoder needs improvements / more testing. Decode OK
3) Untested, with proper narrowing and filtering this may be achievable
over FM-audio
4) My pick for the modulation to use for NBEMS-FM. Tested with less
carriers. Decoder needs improvements


I could use help in the following areas:
* 2-new menu items for NBEMS-FM
* Optimal filters / better RX
* How to reduce inter-carrier interference (optimal filters?)
* Testing on-air
* Implement FEC puncturing (rate 3/4 & 7/8)
* Fix introduced bug: skewed postamble from new 16psk modulator


>From my testing and reading here it seems 6Kbps over FM audio is a real
possibility. This would give 4.5kbps if 3/4 rate FEC is used.
Feedback and ideas appreciated. Will continue coding and testing here.
