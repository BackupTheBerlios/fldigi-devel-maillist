Hi,

At the moment, fldigi uses the external library xmlrpc_c to create a 
XML-RPC server ( http://xmlrpc-c.sourceforge.net/ )

This is fine but I remember that Stelios Bounanos,one of the main 
developers of fldigi, told that he added to fldigi, the sources of 
another xmlrpc server, that is http://xmlrpcpp.sourceforge.net/

As far as I remember (Not sure), his intention was to get rid to 
xmlrpc-c and use instead xmlrpccpp. The consequence would have been:

* Easier to port fldigi to other platforms because at least two external 
libraries are not needed ( xmlrpc-c and curl )
* Possibility to taylor the XMLRPC server to our own needs if necessary 
because all the sources are local.
* Potentially better performance.

What do you think of this ? is it necessary to do this change ?

Thanks
