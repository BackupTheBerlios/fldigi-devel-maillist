Greg Troxel <gdt at work.lexort.com> writes:

> My quick reaction without thinking too much is that it is better to have
> a dependency on an actively-maintained external library than it is to
> copy library sources into fldigi (or any other program).  When there are
> security fixes (or other fixes) in the library, it's then necessary to
> import the new version and have an fldigi release (including stable
> branches).

I should add that packaging systems often patch out this internal use
and make the build use the packaged version, so that when the packaged
version is fixed for a vulnerability then the entire package system is
ok.

Also, I have a bias which views the main consumer of the project's
releases to be packaging systems, and that users run versions that are
produced by packaging systems.  The developer community, on the other
hand, runs versions built by hand from sources obtained from the
revision control system.  Those people have no trouble with external
libraries as long as they are normal and widely packaged.
